## 배경 및 목적
PRD-002에서 구현한 REST API를 활용하는 웹 프론트엔드 페이지를 서버에 내장하여, 웹 브라우저에서 별도 설치 없이 세계 시계를 사용할 수 있도록 한다. 외부 빌드 도구 없이 서버가 HTML을 직접 제공한다.

## 대상 사용자
- 웹 브라우저에서 세계 시계를 확인하고 싶은 사용자
- CLI에 익숙하지 않은 일반 사용자
- 팀 대시보드 등에서 세계 시계를 표시하고 싶은 사용자

## 요구사항
### 기능 요구사항 (Functional)
- FR-1: `GET /` 요청 시 세계 시계 웹 페이지(HTML)를 반환한다. Content-Type은 `text/html`이다.
- FR-2: 웹 페이지는 기존 REST API(`GET /api/clocks`)를 호출하여 모든 도시의 현재 시간을 카드/테이블 형태로 표시하고, 주기적으로(1초마다) 자동 갱신한다.
- FR-3: 웹 페이지에 도시 추가 폼을 제공한다. 도시명과 IANA 타임존을 입력하면 `POST /api/cities`를 호출하여 추가하고 목록을 갱신한다. 에러 시 사용자에게 메시지를 표시한다.
- FR-4: 각 도시 카드에 삭제 버튼을 제공한다. 클릭 시 `DELETE /api/cities/{name}`을 호출하여 삭제하고 목록을 갱신한다.

### 비기능 요구사항 (Non-Functional)
- NFR-1: 외부 CDN이나 빌드 도구 없이 단일 HTML로 제공한다 (서버 내장).
- NFR-2: 반응형 디자인으로 모바일/데스크톱 모두 사용 가능하다.
- NFR-3: 기존 REST API 테스트와 기능에 영향을 주지 않는다.

## 성공 기준
- `world-clock serve` 후 브라우저에서 `http://localhost:3000/` 접속 시 세계 시계 페이지가 표시된다.
- 도시 추가/삭제가 웹 페이지에서 정상 동작한다.

## 범위 외 (Out of Scope)
- 타임존 자동완성/검색
- 사용자 인증
- SSR(Server-Side Rendering) 프레임워크

## 제약 조건
- Rust 바이너리에 HTML을 문자열로 내장 (별도 정적 파일 서빙 불필요)
- 기존 REST API(PRD-002)를 그대로 활용
