## 배경 및 목적
docs/registry.json에 관리되는 추적성 맵(trace_map)을 웹 브라우저에서 인터랙티브 그래프로 시각화한다. PRD(요구사항)부터 SPEC(명세), TC(테스트), CODE(구현)까지의 정방향 추적과, 반대 방향의 역방향 추적을 노드-엣지 그래프로 한눈에 파악할 수 있도록 한다.

## 대상 사용자
- 프로젝트의 요구사항 커버리지를 확인하고 싶은 개발자
- 코드 변경 시 영향 범위를 파악하고 싶은 개발자
- 추적성 현황을 시각적으로 리뷰하고 싶은 팀원

## 요구사항
### 기능 요구사항 (Functional)
- FR-1: `GET /api/trace` 엔드포인트로 registry.json의 trace_map 데이터를 JSON으로 반환한다.
- FR-2: `GET /trace` 요청 시 추적성 그래프를 시각화하는 HTML 페이지를 반환한다. Content-Type은 text/html이다.
- FR-3: 정방향 추적 그래프를 표시한다. PRD → FR → SPEC → TC → 구현파일/심볼 순서로 노드를 계층 배치하고 엣지로 연결한다. 노드 유형별로 색상을 구분한다.
- FR-4: 역방향 추적 그래프를 표시한다. 구현파일/심볼 → TC → SPEC → FR → PRD 순서로 같은 데이터를 역방향 관점에서 탐색할 수 있다. (탭 또는 방향 전환)

### 비기능 요구사항 (Non-Functional)
- NFR-1: 외부 CDN 없이 SVG 기반 자체 그래프 렌더링 (순수 JavaScript).
- NFR-2: 기존 API 및 세계 시계 페이지에 영향 없음.
- NFR-3: 노드 호버 시 상세 정보(제목, 상태 등) 툴팁 표시.

## 성공 기준
- /trace 접속 시 전체 추적 체인이 그래프로 표시된다.
- 정방향/역방향 전환이 가능하다.
- 기존 테스트가 모두 통과한다.

## 범위 외 (Out of Scope)
- 그래프 편집 기능
- registry.json 직접 수정 기능
- 실시간 코드 분석

## 제약 조건
- registry.json을 런타임에 읽어서 제공
- 순수 JavaScript + SVG로 그래프 렌더링 (외부 라이브러리 없음)
