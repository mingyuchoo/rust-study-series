## 배경 및 목적
기존 세계 시계 CLI 앱(PRD-001)의 기능을 HTTP REST API로 제공하여, 웹 브라우저나 외부 클라이언트에서도 세계 시계 기능을 이용할 수 있도록 한다. CLI 앱의 핵심 도메인 로직(clock, config)을 재사용하고, 웹 계층만 추가한다.

## 대상 사용자
- 웹 브라우저에서 세계 시계를 확인하고 싶은 사용자
- REST API를 통해 세계 시계 데이터를 활용하고 싶은 개발자
- 대시보드나 모니터링 도구에 시계를 통합하고 싶은 사용자

## 요구사항
### 기능 요구사항 (Functional)
- FR-1: `GET /api/clocks` 엔드포인트로 저장된 모든 도시의 현재 시간을 JSON으로 반환한다. 도시명, 타임존, 현재 시간, UTC 오프셋을 포함한다.
- FR-2: `POST /api/cities` 엔드포인트로 도시명과 IANA 타임존을 JSON body로 받아 설정에 추가한다. 중복 도시명은 409 Conflict, 잘못된 타임존은 400 Bad Request를 반환한다.
- FR-3: `DELETE /api/cities/{name}` 엔드포인트로 도시를 삭제한다. 존재하지 않는 도시는 404 Not Found를 반환한다.
- FR-4: `GET /api/cities` 엔드포인트로 저장된 도시 목록을 JSON으로 반환한다.
- FR-5: `world-clock serve` CLI 명령으로 웹 서버를 시작한다. `--port` 옵션으로 포트를 지정할 수 있다 (기본값: 3000).

### 비기능 요구사항 (Non-Functional)
- NFR-1: 기존 CLI 기능(PRD-001)은 그대로 유지한다.
- NFR-2: 설정 파일은 CLI와 웹 서비스가 동일한 파일을 사용한다.
- NFR-3: 동시 요청에 대한 안전한 공유 상태 관리 (Mutex 또는 RwLock).
- NFR-4: JSON 응답은 적절한 HTTP 상태 코드와 Content-Type 헤더를 포함한다.

## 성공 기준
- 모든 REST API 엔드포인트가 정상 동작한다.
- 잘못된 요청에 대해 적절한 HTTP 상태 코드와 에러 메시지를 반환한다.
- 기존 CLI 테스트가 모두 통과한다.
