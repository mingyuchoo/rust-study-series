## 배경 및 목적

PRD-002부터 PRD-005까지 웹 API가 CLI 기능과 동등 수준으로 확장되었지만, 임베드된 `index.html`은 여전히 시나리오·리포트 **조회**만 지원한다. 사용자는 `curl`로 POST 요청을 직접 보내야 한다. 본 PRD는 단일 파일 SPA(`index.html`)를 탭 기반으로 재설계하여, 신규 추가된 모든 API를 브라우저에서 직접 사용할 수 있게 한다.

## 대상 사용자
- 브라우저에서 대화형으로 평가 워크플로우를 수행하려는 개발자/QA
- 터미널 사용이 불편한 환경의 팀원

## 요구사항 (기능)
위 추적 표 참조. 8개 FR 각각은 해당 탭에서 **폼 + API 호출 + 결과 렌더링** 3요소를 갖춘다.

## 요구사항 (비기능)
- **NFR-1**: 외부 의존성 없음 — 바닐라 JS/CSS, 바이너리에 `include_str!`로 임베드.
- **NFR-2**: 기존 2-패널 탐색과 비교해 **추가 HTTP 요청만으로** 모든 작업 수행 (페이지 새로고침 없음).
- **NFR-3**: 결과는 JSON pretty-print로 표시, 긴 응답은 스크롤.
- **NFR-4**: 에러 응답은 HTTP 상태 코드 + 메시지로 명확히 표시.

## 성공 기준
- 8개 탭이 모두 동작
- `/api/run`, `/api/scenarios/:d/:id/run`, `/api/tools/:n/invoke`, `/api/agents/:n/execute`, `/api/compare`, `/api/score` 가 UI에서 직접 호출 가능
- 단위 테스트: 임베드된 HTML이 각 탭의 핵심 키워드를 포함

## 범위 외

- 클라이언트 상태 저장/라우팅
