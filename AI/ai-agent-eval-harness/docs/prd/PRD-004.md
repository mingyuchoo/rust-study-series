## 배경 및 목적

PRD-002/003에서 조회 API와 전체 스위트 실행/비교를 노출했다. 그러나 평가 엔진의 **단위 연산들** — 단일 시나리오 실행, 에이전트 직접 호출, 개별 도구 호출, 궤적 채점, 폴트 주입, 궤적 파일 조회 — 은 CLI/라이브러리로만 접근 가능하다. 본 PRD는 `crates/` 4~10번 영역의 기능을 웹에서 **세분화된 단위**로 실행할 수 있도록 7개의 엔드포인트를 추가한다.

## 대상 사용자
- 평가 파이프라인의 특정 단계(도구 호출, 채점, 폴트 주입)만 디버깅하려는 개발자
- 외부 대시보드에서 저장된 궤적을 시각화하려는 팀

## 요구사항

### 기능 요구사항
- **FR-1**: `POST /api/scenarios/:domain/:id/run` body `{"agent":"..."}` → `EvaluationResult`. 내부적으로 `HarnessRunner::run_scenario` 호출.
- **FR-2**: `POST /api/agents/:name/execute` body `{"task":"...","environment":{...}}` → `Trajectory`. `BaseAgent::execute_task` 직접 호출.
- **FR-3**: `POST /api/tools/:name/invoke` body `{"params":{...}}` → `HashMap<String,Value>`. 전 도메인 등록된 ToolRegistry에서 조회 후 호출.
- **FR-4**: `GET /api/golden-sets/:domain/:scenario_id` → `GoldenSetEntry` | 404.
- **FR-5**: `POST /api/score` body `{"trajectory":{...}}` → `EvaluationResult`. `TrajectoryEvaluator::evaluate(&traj, None, None)`.
- **FR-6**: `POST /api/tools/:name/simulate-fault` body `{"params":{...},"config":FaultInjectionConfig}` → 결과 JSON. `FaultInjector::wrap_registry` 후 호출.
- **FR-7**: `GET /api/trajectories` → `Vec<String>` 파일명, `GET /api/trajectories/:name` → 궤적 JSON 원문 또는 404.

### 비기능 요구사항
- **NFR-1**: 모든 실행 API는 `spawn_blocking`으로 감싸 tokio 런타임 블로킹 방지.
- **NFR-2**: 경로 파라미터(`domain`, `id`, `name`, `scenario_id`)는 `is_safe_name` 필터 통과해야 함.
- **NFR-3**: 궤적 디렉토리는 `reporting_trajectories/` 고정(기존 `HarnessRunner` 설정과 일치).
- **NFR-4**: 알 수 없는 에이전트/도구/시나리오는 명시적 에러 반환 (400/404).

## 성공 기준
- 7개 엔드포인트 각각에 대한 단위 테스트 통과
- `POST /api/tools/classify_inquiry/invoke`가 정상 응답
- `POST /api/scenarios/customer_service/cs_001/run`이 passthrough로 EvaluationResult 반환
- `GET /api/trajectories` 가 `run` 실행 후 생성된 파일을 나열
