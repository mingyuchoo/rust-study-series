## 배경 및 목적

PRD-002가 조회 전용 웹 클라이언트(시나리오/리포트 읽기)를 제공했다. 하지만 `crates/` 하위의 12개 크레이트가 제공하는 주요 기능 — 에이전트 레지스트리, 도구 메타데이터, 골든셋, 시나리오 상세, **벤치마크 실행**, **리포트 비교** — 은 여전히 CLI에서만 접근 가능하다. 본 PRD는 이 기능들을 HTTP API로 노출하여 웹에서 평가 워크플로우 전체를 수행할 수 있게 한다.

## 대상 사용자
- 원격에서 벤치마크를 트리거하고 회귀를 감지하려는 QA/DevOps
- 도메인 도구 카탈로그를 탐색하려는 개발자

## 요구사항

### 기능 요구사항 (Functional)
- **FR-1**: `GET /api/agents` → `["passthrough","ppa"]` 등록된 에이전트 이름 배열. (`execution::AgentRegistry`)
- **FR-2**: `GET /api/tools` → `ToolRegistry::get_tools_metadata()` 결과. 도메인별 도구 메타데이터 배열. (`domains::register_*` + `execution-tools::ToolRegistry`)
- **FR-3**: `GET /api/golden-sets` → `eval_data/golden_sets/` 전체 파일의 `GoldenSetFile` 배열. (`data-scenarios::ScenarioLoader::load_all_golden_sets`)
- **FR-4**: `GET /api/scenarios/:domain/:id` → 해당 도메인의 시나리오 하나를 상세 JSON으로 반환. 없으면 404. (`data-scenarios::DomainConfig::scenarios`)
- **FR-5**: `POST /api/run` body `{suite, agent}` → `execution::HarnessRunner::run_suite` 실행, `EvaluationReport` JSON 반환. 블로킹 작업은 `spawn_blocking`.
- **FR-6**: `POST /api/compare` body `{baseline, current, threshold}` → `execution::ReportComparator::compare_files` 실행, `ComparisonResult` JSON 반환.

### 비기능 요구사항 (Non-Functional)
- **NFR-1**: 기존 PRD-002 API(`/`, `/api/scenarios`, `/api/reports`, `/api/reports/:name`)와 완전 호환.
- **NFR-2**: 실행 API(`/api/run`, `/api/compare`)는 비동기 핸들러에서 `spawn_blocking`으로 실행하여 서버 블로킹 방지.
- **NFR-3**: 입력 검증: 에이전트 이름·스위트 이름·파일 경로는 허용 목록/안전 문자만 허용. `compare` 경로는 `reports_dir` 아래로 제한.

## 성공 기준
- 새 6개 엔드포인트가 올바른 상태 코드와 JSON 반환.
- `POST /api/run`이 passthrough 에이전트로 `customer_service` 스위트 실행 시 `EvaluationReport` 반환.
- `POST /api/compare`가 동일 파일 비교 시 `verdict=pass` 반환.

## 범위 외
- 인증/인가
- SSE/WebSocket 기반 실시간 진행률 스트리밍 (다음 PRD)
- 에이전트/도구 **동적 등록** (현재는 빌드 시 컴파일된 것만)

## 제약 조건
- 기존 CLI·TUI·조회 API와 호환
- PPA 에이전트는 `.env` 설정 없을 경우에도 500 없이 `passthrough`만 노출
- 실행 API는 동일 프로세스에서 직렬로만 처리 (동시 실행 락 불필요, 호출자 책임)
