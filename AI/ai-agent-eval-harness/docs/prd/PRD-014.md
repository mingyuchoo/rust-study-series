## 배경 및 목적
현재 eval-harness 웹 UI(`index.html`, `help.html`)는 다크 테마 한 가지로만 고정되어 있어, 밝은 환경에서 작업하는 사용자에게 가독성 문제가 있다. 사용자가 상황에 맞게 라이트/다크 테마를 전환할 수 있도록 하여 접근성과 사용자 경험을 개선한다.

## 대상 사용자
- eval-harness 웹 UI 를 사용하는 AI 에이전트 평가자
- 밝은 사무실 환경/야간 작업 환경을 오가는 개발자

## 요구사항
### 기능 요구사항 (Functional)
- FR-1: 상단 헤더 우측(언어 전환 옆)에 라이트/다크 테마 토글 버튼을 배치한다. 현재 선택된 테마가 시각적으로 구분된다.
- FR-2: 토글 클릭 시 `index.html` 및 `help.html` 의 모든 컴포넌트(header, nav, panel, form, list, button, select, pre, code, badge, note/warn 등)가 선택된 테마의 색상 팔레트에 맞게 일관되게 변경된다.
- FR-3: 선택된 테마는 `localStorage` 의 `theme` 키에 저장되어, 페이지 재로드/재접속 시에도 유지된다. 기본값은 `dark`.

### 비기능 요구사항 (Non-Functional)
- NFR-1: 테마 전환은 페이지 리로드 없이 즉시 반영된다(FOUC 없음).
- NFR-2: 기존 언어 전환(i18n), 탭 네비게이션 등의 동작에 영향을 주지 않는다.
- NFR-3: 두 테마 모두에서 WCAG AA 수준의 텍스트 대비를 유지한다.

## 성공 기준
- 사용자가 토글을 클릭하면 즉시 전체 UI 가 해당 테마로 전환된다.
- 새로고침 후에도 마지막 선택된 테마가 유지된다.
- `index.html` 과 `help.html` 이 동일한 테마 상태를 공유한다.

## 범위 외 (Out of Scope)
- OS 시스템 테마 자동 감지(`prefers-color-scheme`) — 향후 확장 가능하나 이번 범위에는 포함하지 않음.
- 사용자 정의 커스텀 테마.
- desktop(Tauri) 앱의 네이티브 윈도우 테마 연동.

## 제약 조건
- 기존 정적 파일 기반 SPA 구조(`include_str!` 로 번들)를 유지한다.
- 빌드 시스템 변경 없이 순수 HTML/CSS/JS 만으로 구현한다.
- 추가 런타임 의존성(CSS 프레임워크 등) 도입 금지.
