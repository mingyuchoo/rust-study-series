## 배경 및 목적

현재 CLI와 Web API 비교:

| CLI 명령 | 대응 Web API | 상태 |
|---|---|---|
| `list` | `/api/scenarios` + `/api/agents` (분리) | **집계 API 없음** |
| `run --suite --agent --output <file>` | `POST /api/run` | **`--output` 미지원 (디스크 저장 안 함)** |
| `report <file>` | `GET /api/reports/:name` | ✅ |
| `compare baseline current --threshold --output <file>` | `POST /api/compare` | **`--output` 미지원** |
| `tui` | Web SPA 브라우저 | ✅ (동등 기능) |
| `serve` | N/A (서버 자체) | ✅ |

본 PRD는 위 3개 격차를 해소하여 **CLI에서 할 수 있는 모든 기능을 웹에서 수행 가능**하게 한다.

## 요구사항

### 기능 요구사항
- **FR-1**: `POST /api/run` 요청 body에 선택적 `"output": "<filename>"` 필드 지원.
  - 지정 시 `reports_dir/<filename>` 에 `EvaluationReport` JSON 저장.
  - 생략 시 기본 파일명으로 자동 저장 (`evaluation_report_<timestamp>.json`).
  - 저장된 경로는 응답에 `_saved_to` 필드로 포함하거나 새 응답 래퍼 `{report, saved_to}` 사용.
- **FR-2**: `POST /api/compare` 요청 body에 선택적 `"output": "<filename>"` 필드 지원.
  - 지정 시 `reports_dir/<filename>` 에 `ComparisonResult` JSON 저장.
  - 생략 시 파일 저장 없이 응답만 반환 (CLI와 동일 동작).
- **FR-3**: `GET /api/list` → `{ "domains": [DomainSummary], "agents": [String] }` 단일 페이로드.

### 비기능 요구사항
- **NFR-1**: `output` 파라미터는 `is_safe_name` 통과 필수 (경로 순회 차단).
- **NFR-2**: 기존 요청 형식과 역호환 (`output` 없는 기존 호출은 그대로 동작).
- **NFR-3**: 새 라우트 `/api/list` 는 기존 `/api/scenarios`, `/api/agents` 와 공존.

## 성공 기준
- 3개 FR 각각 단위 테스트 통과
- `curl POST /api/run -d '{"suite":"customer_service","agent":"passthrough","output":"my.json"}'` → `reports_dir/my.json` 생성
- `curl GET /api/list` → domains + agents 필드 모두 포함

## 범위 외
- CLI 의 `--scenarios-dir`, `--output-dir` 등 **경로 오버라이드**는 제공하지 않음 (경로 순회 방지 및 서버-레벨 설정 존중)
- TUI 의 대화형 조작 자체

## 제약 조건
- 기존 API 엔드포인트 동작 변경 금지 (추가만 허용)
- `eval-harness` 크레이트만 수정
