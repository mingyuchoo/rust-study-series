## 배경 및 목적
현재 `eval_data/eval_scenarios`, `eval_data/golden_sets` 경로가 CLI 진입점(`crates/eval-harness/src/main.rs`)의 5 곳에 `default_value` 로 하드코딩되어 있고, 데스크톱 진입점(`desktop/src/main.rs`)에서도 워크스페이스 루트 기준으로 직접 결합되어 있다. 평가 데이터셋을 다른 디렉토리(예: 사내 공유 드라이브, 별도 데이터셋 리포지토리)에 두는 사용자가 매번 CLI 인자를 5 개씩 지정해야 하고, 데스크톱 앱은 아예 변경이 불가능하다.

설정 파일을 통해 한 번만 경로를 지정해두면 모든 진입점이 동일한 데이터셋을 사용하도록 한다. 또한 데이터셋 경로를 상대 경로로 적어 리포지토리 간 이식성을 확보한다.

## 대상 사용자
- 평가 데이터셋을 별도 디렉토리에서 관리하는 평가자/연구자
- 같은 코드베이스에서 여러 데이터셋을 전환해 가며 평가하는 개발자
- 데스크톱 앱 사용자

## 요구사항

### 기능 요구사항 (Functional)
- FR-1: 프로젝트 루트(또는 사용자가 지정한 위치)의 `eval-harness.toml` 파일에 다음 형식의 설정을 읽는다.
  ```toml
  [data]
  scenarios_dir   = "eval_data/eval_scenarios"
  golden_sets_dir = "eval_data/golden_sets"
  ```
  키가 누락되면 해당 키에 대해서만 내장 기본값으로 fallback 한다.
- FR-2: 설정 파일의 경로 값이 절대 경로(`/`/`C:\\` 등)이면 그대로, 상대 경로이면 **설정 파일이 위치한 디렉토리** 를 기준으로 해석된다. 설정 파일이 없는 상태로 환경변수만 사용하는 경우, 환경변수의 상대 경로는 현재 작업 디렉토리(CWD) 기준으로 해석된다.
- FR-3: 환경변수 `EVAL_HARNESS_SCENARIOS_DIR`, `EVAL_HARNESS_GOLDEN_SETS_DIR` 값이 설정되어 있으면 설정 파일의 값보다 우선한다.
- FR-4: CLI 의 `--scenarios-dir`, `--golden-sets-dir` 옵션은 환경변수보다도 우선한다. 즉 우선순위는 `CLI > ENV > config file > 내장 기본값`. CLI 의 `default_value` 는 제거되고 옵션이 없을 때만 하위 단계가 평가된다.
- FR-5: 설정 파일/환경변수/CLI 인자가 모두 없으면 기존 동작과 동일하게 `eval_data/eval_scenarios`, `eval_data/golden_sets` 를 사용한다(현재 작업 디렉토리 기준).
- FR-6: `desktop/src/main.rs` 도 `crates/eval-harness` 의 설정 해석 로직을 호출하여 `eval_data/` 경로 하드코딩을 제거한다. 데스크톱은 워크스페이스 루트(`CARGO_MANIFEST_DIR/..`)를 기본 검색 위치로 사용한다.

### 비기능 요구사항 (Non-Functional)
- NFR-1: 설정 파일이 없거나 일부 키가 누락되어도 기존 사용자 경험이 깨지지 않는다(silently fall back).
- NFR-2: 설정 파일에 잘못된 TOML 문법이 있으면 명확한 에러 메시지(파일 경로 + 파싱 오류)와 함께 종료 코드 1 로 종료한다.
- NFR-3: 신규 런타임 의존성은 추가하지 않는다(`toml` 크레이트는 Rust 생태계 표준이며 워크스페이스에 이미 사용 중이거나 추가 가능 — 새로 추가될 경우 `crates/eval-harness` 한정).

## 성공 기준
- 사용자가 `eval-harness.toml` 한 파일만 두고 `cargo run -- list` 를 실행하면, CLI 인자 없이도 설정된 디렉토리에서 시나리오를 로드한다.
- 환경변수 하나만 export 해도 설정 파일/내장 기본값보다 우선 적용된다.
- desktop 앱이 동일 설정 파일을 인식하여 동일한 데이터셋을 사용한다.
- 잘못된 TOML 은 명확한 에러로 실패한다.

## 범위 외 (Out of Scope)
- `reporting_logs/`, `reporting_trajectories/` 등 출력 디렉토리의 외부 설정화. (별도 PRD 로 분리 가능)
- 시나리오 파일 단위의 화이트리스트/필터링.
- 설정 파일의 hot reload.
- 웹 UI 상에서 설정 값을 편집하는 기능.

## 제약 조건
- 기존 CLI 옵션 이름(`--scenarios-dir`, `--golden-sets-dir`)은 그대로 유지한다(하위 호환).
- 기존 테스트(특히 `crates/eval-harness/src/web/api*.rs` 의 `workspace_scenarios()` 헬퍼)는 워크스페이스 루트 기반의 자체 경로 해석을 유지하여 CI 격리성을 보장한다.
- 데스크톱 앱은 컴파일 타임 워크스페이스 루트를 기본 검색 디렉토리로 사용하므로, 인스톨 디렉토리 기반 해석으로의 전환은 별도 작업으로 둔다.
