## 배경 및 목적
Tauri(WebKit2GTK) 로 실행하면 `<select>` 의 닫힌 상태는 커스텀 CSS 가 적용되지만, **드롭다운 팝업 리스트**는 OS 네이티브 위젯을 사용해 **밝은 배경 + 어두운 텍스트**로 렌더된다. 다크 테마 SPA 와 시각적으로 충돌해 가독성이 떨어진다.

또한 닫힌 상태의 `<select>` 도 브라우저 기본 화살표가 작고 주변 다크 UI 와 이질감이 있어, 커스텀 화살표와 명시적 높이·패딩이 필요하다.

## 요구사항

### 기능 요구사항
- **FR-1**: `:root { color-scheme: dark; }` 선언. 브라우저가 네이티브 위젯(스크롤바, select 팝업 등)을 다크 모드로 렌더한다. WebKit·Chromium·Firefox 모두 지원.
- **FR-2**: `select option { background: #1e1e1e; color: #e0e0e0; }` 로 팝업 내 항목의 배경/텍스트를 명시. 일부 엔진(Chromium)은 `option` 에 CSS 적용이 가능하며, 나머지 엔진은 FR-1 로 폴백.
- **FR-3**: 닫힌 `<select>` 에:
  - `appearance: none` + `-webkit-appearance: none`
  - 오른쪽에 인라인 SVG data-URL 화살표를 `background-image` 로 삽입
  - 우측 패딩을 확보 (화살표 공간)
  - 라인 높이·패딩 통일

### 비기능 요구사항
- **NFR-1**: 기존 폼 레이아웃/버튼/입력의 높이/정렬에 영향 없음 (6px 상하 패딩 유지)
- **NFR-2**: `<select>` focus 링을 유지 (접근성)

## 성공 기준
- Tauri 창에서 `<select>` 열 때 팝업 리스트가 다크 배경 + 밝은 텍스트로 표시
- 닫힌 `<select>` 에 커스텀 화살표가 노출
- 단위 테스트: `color-scheme: dark`, `select option`, `appearance: none` 키워드 포함

## 범위 외
- `<select>` 를 순수 JS/CSS 커스텀 컴포넌트로 완전 재구현
- 검색·필터링 가능한 combobox 위젯
