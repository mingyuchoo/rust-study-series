## 배경 및 목적

현재 `eval-harness`는 CLI 서브커맨드(`list`, `run`, `report`, `compare`)로만 동작한다. 사용자가 시나리오 목록과 저장된 리포트를 빠르게 탐색하려면 여러 명령을 반복 실행해야 한다. 대화형 TUI(Terminal User Interface)를 추가하여 시나리오와 리포트를 한 화면에서 탐색 가능하게 한다.

## 대상 사용자

- AI 에이전트 평가 결과를 반복적으로 조회하는 개발자
- 터미널 환경에서 작업하며 GUI가 없는 원격 서버 사용자

## 요구사항

### 기능 요구사항 (Functional)
- **FR-1**: 사용자는 `cargo run -- tui` 명령으로 TUI 모드를 실행할 수 있다.
- **FR-2**: TUI는 `eval_data/scenarios/` 디렉토리의 시나리오 목록을 표시하고, ↑/↓/j/k 키로 항목을 선택할 수 있다.
- **FR-3**: TUI는 `reporting_logs/` 디렉토리의 저장된 리포트 파일 목록을 표시하며, Tab 키로 시나리오/리포트 패널을 전환할 수 있다.
- **FR-4**: 사용자는 `q` 또는 `Esc` 키로 TUI를 종료할 수 있다.

### 비기능 요구사항 (Non-Functional)
- **NFR-1**: TUI 초기 렌더링은 1초 이내에 완료되어야 한다.
- **NFR-2**: 터미널 raw 모드 전환/복원은 정상 종료 및 패닉 시 모두 복원되어야 한다.

## 성공 기준

- `cargo run -- tui` 실행 시 에러 없이 TUI가 렌더링된다.
- 시나리오 목록과 리포트 목록이 표시되고 키보드로 탐색 가능하다.
- `q` 키 입력 후 터미널이 정상 상태로 복원된다.

## 범위 외 (Out of Scope)

- TUI 내부에서 벤치마크 직접 실행 (본 PRD는 **조회 전용**)
- 리포트 파일 상세 내용 렌더링 (다음 PRD로 분리)
- 마우스 이벤트 지원

## 제약 조건

- 기존 CLI 서브커맨드와 호환성 유지 (`run`, `list`, `report`, `compare`는 그대로 동작)
- Rust nightly 툴체인 사용
- 크로스 플랫폼(Linux/macOS) 지원 필요
