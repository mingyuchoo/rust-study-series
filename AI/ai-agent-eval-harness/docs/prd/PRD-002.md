## 배경 및 목적

CLI/TUI만으로는 비개발자나 원격 팀이 평가 결과를 공유하기 어렵다. HTTP 서버와 브라우저 기반 간단한 SPA를 제공하여 링크 공유와 원격 조회를 가능하게 한다.

## 대상 사용자
- 팀 내 PM/QA 등 비개발자
- 원격 서버에 배포된 eval-harness 결과를 공유하려는 DevOps

## 요구사항

### 기능 요구사항 (Functional)
- **FR-1**: `cargo run -- serve [--port 8080] [--scenarios-dir ...] [--reports-dir ...]` 로 서버 실행.
- **FR-2**: `GET /api/scenarios` → 모든 도메인 YAML을 파싱한 `[{name, description, scenarios: [{id,name,difficulty}]}]`.
- **FR-3**: `GET /api/reports` → `reporting_logs/` 내 `.json` 파일명 정렬 배열.
- **FR-4**: `GET /api/reports/:name` → 해당 파일의 JSON 원문 반환. 없으면 404.
- **FR-5**: `GET /` → 임베드된 정적 `index.html`. 시나리오/리포트를 사이드바로, 선택 시 상세를 본문에 표시.

### 비기능 요구사항 (Non-Functional)
- **NFR-1**: 기동 2초 이내.
- **NFR-2**: 경로 순회 공격 방지 (리포트 name 에 `/`, `..` 금지).

## 성공 기준
- `cargo run -- serve` → `http://127.0.0.1:8080` 접속 시 SPA 렌더.
- 4개 API가 각각 올바른 상태 코드와 페이로드 반환.
