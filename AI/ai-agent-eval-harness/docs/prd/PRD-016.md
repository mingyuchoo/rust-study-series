## 배경 및 목적
현재 `PpaAgent` 의 루프 상한값(`max_iterations`)과 조기 종료 임계값(`early_stop_threshold`) 은 `EvaluationConfig::default()` 에 하드코딩된 `3` 이다. 태스크 복잡도에 따라 이 값을 조정하려면 소스 코드를 고쳐 재컴파일해야 한다.

PRD-015 에서 도입한 `eval-harness.toml` 에 `[evaluation]` 섹션을 추가하여 재컴파일 없이 실행 시점에 조정할 수 있게 한다.

## 대상 사용자
- 복잡한 태스크에 대해 반복 횟수를 5~10 회로 늘리고 싶은 평가자
- 빠른 반복 테스트를 위해 2 회로 줄이고 싶은 개발자
- 에이전트 동작을 조정하되 코드 변경 권한이 없는 사용자

## 요구사항

### 기능 요구사항
- **FR-1**: `eval-harness.toml` 에 아래 섹션을 지원한다.
  ```toml
  [evaluation]
  max_iterations        = 5
  early_stop_threshold  = 3
  ```
  각 키는 양의 정수여야 한다.
- **FR-2**: 섹션 전체 또는 개별 키가 없으면 해당 키만 내장 기본값으로 fallback 한다. 이 동작은 `[data]` 섹션의 부분 누락 처리와 동일하다.
- **FR-3**: `eval-harness` CLI 와 `serve` 웹 서버(`build_agent_registry`) 모두 동일한 해석 함수를 호출하여 `PpaAgent::new` 에 전달한다.

### 비기능 요구사항
- **NFR-1**: 잘못된 값(0, 음수, 문자열 등)은 TOML 파싱 단계 또는 해석 단계에서 명확한 에러로 실패한다.
- **NFR-2**: 기존 `[data]` 섹션과 동일 파일을 공유하므로 I/O 를 중복 수행하지 않도록 파일을 한 번만 읽는다.

## 성공 기준
- `[evaluation] max_iterations = 5` 설정 후 `cargo run -- run --suite financial --agent ppa` 실행 시 각 시나리오가 최대 5 회까지 반복.
- 설정 파일 없이 실행 시 기존과 동일하게 최대 3 회.
- 웹 UI 의 "Agents" 탭 실행도 동일한 값이 적용됨.

## 범위 외
- `[evaluation]` 외 다른 `EvaluationConfig` 필드(`log_dir`, `trajectory_dir`, `enable_llm_judge` 등) 노출
- CLI 옵션/환경변수로 `[evaluation]` 을 override 하는 경로 (추후 필요시)

## 제약 조건
- 기존 `EvaluationConfig` 구조체 변경 없음 (필드 추가/삭제 금지)
- `eval-harness.toml` 의 기존 `[data]` 섹션과 완전 호환
