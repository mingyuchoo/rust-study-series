## 배경 및 목적

PRD-006에서 7-탭 SPA를 도입했지만, 첫 방문자에게는 각 탭이 어떤 API를 사용하고 어떤 워크플로우를 수행할 수 있는지 직관적으로 드러나지 않는다. CLI/Web/TUI 전체 기능을 한눈에 볼 수 있는 **독립된 사용안내 페이지**를 제공하여 온보딩 비용을 낮춘다.

## 요구사항

### 기능 요구사항
- **FR-1**: `GET /help` → 별도의 임베드 HTML 페이지(`help.html`) 반환. 내용:
  - 프로젝트 개요
  - 7개 탭별 사용 흐름 (Run / Scenarios / Tools / Agents / Reports / Trajectories / Goldens)
  - 주요 REST 엔드포인트 목록 + curl 예시
  - CLI 명령 대응 표
  - 주의사항 (인증 없음, 경로 순회 차단, passthrough vs ppa)
- **FR-2**: `index.html` 상단 헤더 우측에 "사용안내" 링크/버튼 추가. `target="_blank"` 로 `/help` 새 탭 오픈.

### 비기능 요구사항
- **NFR-1**: 외부 의존성 없음. `include_str!` 로 임베드.
- **NFR-2**: 메인 SPA와 동일한 다크 테마.

## 성공 기준
- 서버 기동 후 `http://127.0.0.1:8080/help` 접속 시 가이드 페이지 렌더
- 메인 SPA 헤더의 "사용안내" 버튼 클릭 시 새 탭에서 `/help` 열림
- 단위 테스트: help 본문이 필수 키워드(Run/Scenarios/Tools/Agents/Reports/Trajectories/Goldens, `/api/run`, `curl`) 포함
