eval-harness — 사용안내

← 메인 화면으로

1. 개요

ai-agent-eval-harness 는 Rust 로 구현된 AI 에이전트 평가 하네스입니다. 다양한 도메인 시나리오에서 에이전트의 도구 호출 정확도, 성능, 회귀 여부를 자동으로 평가합니다. 본 웹 클라이언트는 CLI 가 제공하는 모든 기능을 브라우저에서 직접 수행할 수 있도록 합니다.

빠른 시작

# 1) 빌드
cargo build

# 2) 웹 서버 기동 (기본 포트 8080)
cargo run -- serve

# 3) 브라우저에서 http://127.0.0.1:8080 접속
에이전트 선택: 기본 에이전트 passthrough 는 환경 설정 없이 즉시 사용 가능합니다. Azure OpenAI 기반 ppa 에이전트를 쓰려면 프로젝트 루트에 .env 파일을 만들고 AZURE_OPENAI_* 키를 설정하세요.

2. 화면 구성 (7개 탭)

메인 SPA 는 상단 탭 바로 구성됩니다. 각 탭은 관련 REST API 를 호출하여 작업 결과를 현재 화면에서 보여줍니다.

Run 벤치마크 스위트 실행

드롭다운에서 suite(all / customer_service / financial)와 agent를 선택하고, 선택적으로 출력 파일명을 지정한 뒤 Run 버튼을 누릅니다.

  • 호출: POST /api/run body {"suite","agent","output?"}
  • 결과: EvaluationReport JSON이 표시되고, reports_dir/ 에 파일로 저장됩니다 (기본: evaluation_report_<timestamp>.json).

Scenarios 시나리오 탐색 + 단일 실행

좌측 트리에서 도메인별 시나리오를 탐색합니다. 시나리오를 선택하면 우측에 상세 정보가 표시되고, 에이전트를 선택한 뒤 Run this scenario 버튼으로 단일 시나리오만 실행할 수 있습니다.

  • 호출: GET /api/list, GET /api/scenarios/:domain/:id, POST /api/scenarios/:domain/:id/run

Tools 도구 직접 호출 + 폴트 주입

좌측 목록에서 도구를 선택하면 우측에 파라미터 스키마가 표시됩니다. Parameters (JSON) 텍스트 에어리어에 JSON 을 입력하고 Invoke 버튼으로 정상 호출하거나, Simulate fault 버튼으로 폴트 주입 후 호출합니다 (seed, failure_rate 지정 가능).

  • 호출: GET /api/tools, POST /api/tools/:name/invoke, POST /api/tools/:name/simulate-fault

Agents 에이전트 직접 호출

에이전트를 선택하고, 자유 태스크 문장과 선택적 environment JSON 을 입력한 뒤 Execute 로 실행합니다. 시나리오 없이 임의 태스크를 실행해볼 때 유용합니다.

  • 호출: POST /api/agents/:name/execute body {"task","environment?"}
  • 결과: Trajectory JSON (PPA steps 포함)

Reports 저장된 리포트 조회 + 비교

좌측 목록에서 저장된 리포트 JSON 을 선택하면 우측에 내용이 표시됩니다. 하단 Compare 패널에서 두 리포트를 골라 임계값과 함께 비교합니다.

  • 호출: GET /api/reports, GET /api/reports/:name, POST /api/compare

Trajectories 저장된 궤적 조회 + 채점

스위트/시나리오 실행 시 저장되는 trajectory JSON 파일들을 목록에서 확인합니다. 선택한 궤적을 Score this trajectory 버튼으로 다시 채점할 수 있습니다.

  • 호출: GET /api/trajectories, GET /api/trajectories/:name, POST /api/score

Goldens 골든셋 조회

전체 골든셋 파일을 한 번에 조회하거나, domain + scenario_id 를 입력해 단일 기준 엔트리만 가져올 수 있습니다.

  • 호출: GET /api/golden-sets, GET /api/golden-sets/:domain/:scenario_id

실행 API (POST, spawn_blocking)

메서드경로요청 body
POST/api/run{"suite","agent","output?"}
POST/api/compare{"baseline","current","threshold?","output?"}
POST/api/scenarios/:d/:id/run{"agent"}
POST/api/agents/:name/execute{"task","environment?"}
POST/api/tools/:name/invoke{"params"}
POST/api/tools/:name/simulate-fault{"params","config"}
POST/api/score{"trajectory"}

curl 예시

# 벤치마크 실행
curl -X POST http://127.0.0.1:8080/api/run \
  -H 'content-type: application/json' \
  -d '{"suite":"customer_service","agent":"passthrough","output":"my.json"}'

# 단일 도구 호출
curl -X POST http://127.0.0.1:8080/api/tools/classify_inquiry/invoke \
  -H 'content-type: application/json' \
  -d '{"params":{"inquiry_text":"환불 요청","customer_id":"C1"}}'

# 단일 시나리오 실행
curl -X POST http://127.0.0.1:8080/api/scenarios/customer_service/cs_001/run \
  -H 'content-type: application/json' \
  -d '{"agent":"passthrough"}'

# 리포트 비교
curl -X POST http://127.0.0.1:8080/api/compare \
  -H 'content-type: application/json' \
  -d '{"baseline":"a.json","current":"b.json","threshold":5.0}'

# 에이전트 직접 실행
curl -X POST http://127.0.0.1:8080/api/agents/passthrough/execute \
  -H 'content-type: application/json' \
  -d '{"task":"hello"}'

5. 주의사항

인증/인가 없음: 이 서버는 기본적으로 인증이 없습니다. 로컬/내부망 전용이며, 외부 노출 시 반드시 리버스 프록시에서 인증 레이어를 추가하세요.
경로 순회 차단: 모든 파일명 파라미터(리포트/궤적/output 등)는 /, \\, .., 빈 문자열을 거부합니다. 지정한 디렉토리 내부 파일만 접근 가능합니다.
블로킹 작업: /api/run, /api/compare, /api/scenarios/:d/:id/run, /api/agents/:n/execute 등 동기 로직은 tokio::task::spawn_blocking 으로 감싸 서버 블로킹을 방지합니다.
