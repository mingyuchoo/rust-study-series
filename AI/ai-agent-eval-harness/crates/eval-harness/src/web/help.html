1. 개요

ai-agent-eval-harness 는 Rust 로 구현된 AI 에이전트 평가 하네스입니다. 다양한 도메인 시나리오에서 에이전트의 도구 호출 정확도, 성능, 회귀 여부를 자동으로 평가합니다. 본 웹 클라이언트는 CLI 가 제공하는 모든 기능을 브라우저에서 직접 수행할 수 있도록 합니다.

빠른 시작

# 1) 빌드
cargo build

# 2) 웹 서버 기동 (기본 포트 8080)
cargo run -- serve

# 3) 브라우저에서 http://127.0.0.1:8080 접속
에이전트 선택: 기본 에이전트 passthrough 는 환경 설정 없이 즉시 사용 가능합니다. Azure OpenAI 기반 ppa 에이전트를 쓰려면 프로젝트 루트에 .env 파일을 만들고 AZURE_OPENAI_* 키를 설정하세요.

2. 화면 구성 (7개 탭)

메인 SPA 는 상단 탭 바로 구성됩니다. 각 탭은 관련 REST API 를 호출하여 작업 결과를 현재 화면에서 보여줍니다.

Run 평가 시나리오 실행

평가 시나리오와 에이전트를 선택하고 선택적으로 출력 파일명을 지정한 뒤 Run 버튼을 누릅니다.

  • 호출: POST /api/run body {"eval_scenario","agent","output?"}

Scenarios 시나리오 탐색 + 단일 실행

좌측 트리에서 시나리오를 선택하고 에이전트를 지정해 단일 시나리오만 실행합니다.

  • 호출: GET /api/list, POST /api/scenarios/:d/:id/run

Tools 도구 직접 호출 + 폴트 주입

도구 선택 후 Parameters JSON 을 입력하여 정상 호출하거나 폴트 시뮬레이션을 수행합니다.

  • 호출: POST /api/tools/:name/invoke, POST /api/tools/:name/simulate-fault

Agents 에이전트 직접 호출

자유 태스크 + 선택적 environment JSON 을 에이전트에 직접 전달합니다. 결과는 Trajectory JSON 으로 표시됩니다.

  • 호출: POST /api/agents/:name/execute

Reports 리포트 조회 + 비교

저장된 리포트를 선택해 내용을 확인하고, 두 리포트를 선택해 회귀 여부를 비교합니다.

  • 호출: GET /api/reports[/:name], POST /api/compare

Trajectories 궤적 조회 + 채점

저장된 궤적 파일을 조회하고 다시 채점할 수 있습니다.

  • 호출: GET /api/trajectories[/:name], POST /api/score

Goldens 골든셋 조회

전체 골든셋 파일 또는 domain + scenario_id 로 단일 엔트리를 조회합니다.

  • 호출: GET /api/golden-sets[/:d/:sid]

1. Overview

ai-agent-eval-harness is an AI agent evaluation harness implemented in Rust. It automatically evaluates tool-calling accuracy, performance, and regression of AI agents across various domain scenarios. This web client makes every CLI feature accessible directly from your browser.

Quick start

# 1) Build
cargo build

# 2) Start web server (default port 8080)
cargo run -- serve

# 3) Open http://127.0.0.1:8080
Agent selection: the default passthrough agent works with no setup. To use the Azure OpenAI-backed ppa agent, create a .env file in the project root with AZURE_OPENAI_* variables.

2. UI layout (7 tabs)

The main SPA is organised as a top tab bar. Each tab calls a related REST API and renders the result inline.

Run Run eval scenario

Pick an eval scenario and agent, optionally set an output filename, then click Run.

  • Calls: POST /api/run body {"eval_scenario","agent","output?"}

Scenarios Browse and run a single scenario

Select a scenario from the tree on the left, pick an agent, and run only that one scenario.

  • Calls: GET /api/list, POST /api/scenarios/:d/:id/run

Tools Direct tool invocation and fault injection

After selecting a tool, edit the Parameters JSON and invoke it, or simulate a fault-injected call.

  • Calls: POST /api/tools/:name/invoke, POST /api/tools/:name/simulate-fault

Agents Direct agent execute

Send a free-form task and optional environment JSON directly to an agent. The result is a Trajectory JSON.

  • Calls: POST /api/agents/:name/execute

Reports View reports and compare

Open a stored report to inspect its contents, or pick two reports to compare for regressions.

  • Calls: GET /api/reports[/:name], POST /api/compare

Trajectories Browse trajectories and re-score

Browse saved trajectory files and re-score any of them with the evaluator.

  • Calls: GET /api/trajectories[/:name], POST /api/score

Goldens Inspect golden sets

List all golden-set files or fetch a single entry by domain + scenario_id.

  • Calls: GET /api/golden-sets[/:d/:sid]

5. Notes

No authentication: this server ships with no auth. Local/internal use only; when exposing publicly, put an auth layer in a reverse proxy.
Path-traversal blocked: all filename parameters reject /, \\, .., and empty strings.
Blocking work: synchronous logic is wrapped in tokio::task::spawn_blocking to avoid blocking the server.
