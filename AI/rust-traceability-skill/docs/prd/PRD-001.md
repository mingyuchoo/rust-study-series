## 배경 및 목적
블로그 서비스의 핵심인 게시글 관리 기능을 REST API로 제공한다. Axum 웹 프레임워크를 사용하여 타입 안전한 라우팅과 비동기 처리를 지원하며, 인메모리 저장소를 사용하여 외부 의존성 없이 동작한다.

## 대상 사용자
블로그 프론트엔드 개발자 또는 API 클라이언트 사용자

## 요구사항
### 기능 요구사항 (Functional)
- FR-1: 게시글 생성 - 제목과 본문을 입력받아 새 게시글을 생성하고, 고유 ID와 생성 시각을 자동 부여하여 반환한다.
- FR-2: 게시글 단건 조회 - ID로 특정 게시글을 조회한다. 존재하지 않는 ID로 조회 시 404를 반환한다.
- FR-3: 게시글 목록 조회 - 저장된 모든 게시글을 목록으로 반환한다.
- FR-4: 게시글 수정 - ID로 특정 게시글의 제목과 본문을 수정한다. 존재하지 않는 ID로 수정 시 404를 반환한다.
- FR-5: 게시글 삭제 - ID로 특정 게시글을 삭제한다. 존재하지 않는 ID로 삭제 시 404를 반환한다.

### 비기능 요구사항 (Non-Functional)
- NFR-1: JSON 형식의 REST API로 제공한다.
- NFR-2: 비동기(async) 처리를 지원한다.
- NFR-3: 순수 로직(도메인)과 부수 효과(HTTP 핸들러, 저장소)를 분리한다.

## 성공 기준
- 모든 CRUD 엔드포인트가 정상 동작한다.
- 존재하지 않는 리소스 접근 시 적절한 에러 응답(404)을 반환한다.
- 모든 테스트 케이스가 통과한다.
